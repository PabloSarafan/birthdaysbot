# 🎂 Birthday Reminder Bot

Telegram бот-напоминалка о днях рождения. Автоматически напоминает о предстоящих днях рождения за 7, 3 и 1 день.

## ✨ Возможности

- 📝 **Добавление дней рождения** через удобный диалоговый интерфейс
- 📋 **Просмотр всех записей** с сортировкой по близости даты
- ✏️ **Редактирование** существующих записей
- 🗑️ **Удаление** ненужных записей
- 🔔 **Автоматические уведомления** за 7, 3 и 1 день до дня рождения
- 💾 **Надежное хранение** в SQLite базе данных
- 👥 **Многопользовательский режим** - каждый пользователь видит только свои записи

## 📋 Требования

- Python 3.7+ (рекомендуется 3.11-3.13)
- Токен Telegram бота (получить у [@BotFather](https://t.me/BotFather))

**Примечание**: Для Python 3.13+ проект включает совместимый модуль `imghdr.py`, так как этот модуль был удален из стандартной библиотеки.

## 🚀 Установка и запуск локально

### 1. Клонируйте репозиторий:
```bash
git clone <your-repo-url>
cd birthdaysbot
```

### 2. Создайте виртуальное окружение и активируйте его:
```bash
python3 -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

### 3. Установите зависимости:
```bash
pip install -r requirements.txt
```

### 4. Настройте переменные окружения

- **Токен бота**: создайте `apibot.env` и добавьте `BOT_TOKEN=ваш_токен_от_BotFather`  
  (или используйте `.env` на основе `env.example`).
- **OpenAI (опционально)**: для генерации поздравлений скопируйте `openai.env.example` в `openai.env` и укажите ваш ключ:
  ```bash
  cp openai.env.example openai.env
  # Отредактируйте openai.env: OPENAI_API_KEY=sk-...
  ```

### 5. Запустите бота:
```bash
python bot.py
```

### 🚀 Деплой на продакшен (CapRover и др.)

На сервере Telegram должен отправлять обновления по **webhook** (по умолчанию бот использует long polling). Задайте переменные окружения:

- **`WEBHOOK_URL`** — полный HTTPS-адрес вашего бота, куда Telegram будет слать обновления (например: `https://birthdaybot.ваш-домен.com` или `https://ваш-домен.com/webhook`).
- **`PORT`** — порт, на котором слушает приложение (на CapRover обычно задаётся автоматически).

Если заданы и `WEBHOOK_URL`, и `PORT`, бот запускается в режиме webhook и сам вызывает `set_webhook`. Иначе используется long polling (как локально).

**Важно:** должен быть только один способ получения обновлений: либо webhook на проде, либо polling локально. Не запускайте два экземпляра с одним и тем же токеном.

#### Чек-лист для CapRover (бот не отвечает)

1. **App Configs** → добавьте переменные окружения (не копируйте в репо `apibot.env` — в Docker его нет):
   - `BOT_TOKEN` — токен от @BotFather
   - `WEBHOOK_URL` — **ровно** ваш публичный URL, например: `https://birthdaybot.sarafannikov.work` (без слэша в конце)
   - `PORT` — обычно CapRover подставляет сам (80); если нет — укажите `80` (как в HTTP Settings → Container HTTP Port)
   - Для работы **генерации поздравлений**: `OPENAI_API_KEY` — ключ с [platform.openai.com](https://platform.openai.com/api-keys); опционально `OPENAI_MODEL` (по умолчанию `gpt-4o-mini`)

2. После сохранения конфига **сделайте перезапуск приложения** (Deployment → Restart).

3. Откройте **Logs** и убедитесь, что есть строка:  
   `Запуск в режиме webhook: https://birthdaybot.sarafannikov.work (порт 80, path ...)`  
   Если видите «Webhook снят, используется long polling» — значит `WEBHOOK_URL` или `PORT` не подхватились; проверьте App Configs.

4. Проверка webhook в Telegram: откройте в браузере или через curl:  
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getWebhookInfo`  
   Подставьте токен **без пробелов и переносов** (одна строка: `bot` + токен + `/getWebhookInfo`).  
   - В ответе должно быть `"ok":true` и `"url":"https://birthdaybot.sarafannikov.work/"`.  
   - Если пришло `{"ok":false,"error_code":404}` — проверьте URL: токен скопирован целиком, нет лишнего слэша или пробела.  
   - Если открывали **свой** домен (https://birthdaybot.sarafannikov.work) в браузере и получили 404 — это нормально: бот принимает только POST от Telegram, GET в браузере не обрабатывается.

5. При тесте прода **остановите** локальный запуск бота (`python bot.py`), иначе обновления может забирать только один экземпляр.

## 🤖 Получение токена бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям: придумайте имя и username для бота
4. BotFather отправит вам токен — скопируйте его в `apibot.env` (или `.env`) в переменную `BOT_TOKEN`

## 📖 Команды бота

### `/start`
Приветствие и описание возможностей бота

### `/add`
Добавление нового дня рождения через диалог:
1. Введите ФИО человека (например: Иванов Иван Иванович)
2. Введите дату рождения в формате ДД.ММ.ГГГГ (например: 15.03.1990)
3. Бот сохранит данные и подтвердит добавление

### `/list`
Показать все дни рождения:
- Отсортированы по близости даты (ближайшие сверху)
- Показывает сколько дней осталось до каждого дня рождения
- Если день рождения сегодня - показывает "СЕГОДНЯ!"

### `/delete`
Удаление записи:
1. Бот покажет пронумерованный список
2. Введите номер записи для удаления
3. Запись будет удалена

### `/edit`
Редактирование записи:
1. Бот покажет пронумерованный список
2. Введите номер записи для редактирования
3. Введите новое ФИО
4. Введите новую дату рождения
5. Запись будет обновлена

### `/check`
Проверить уведомления вручную:
- Полезно для тестирования
- Отправляет все подходящие уведомления прямо сейчас
- Не нужно ждать 09:00

### `/cancel`
Отменить текущую операцию (добавление/редактирование/удаление)

## 🔔 Автоматические уведомления

Бот автоматически проверяет дни рождения каждый день в **09:00 по московскому времени (MSK)** и отправляет уведомления:

- **За 7 дней**: "🎂 Не забудь поздравить [ФИО] через 7 дней (ДД.ММ.ГГГГ)!"
- **За 3 дня**: "🎂 Не забудь поздравить [ФИО] через 3 дня (ДД.ММ.ГГГГ)!"
- **За 1 день**: "🎂 Не забудь поздравить [ФИО] завтра (ДД.ММ.ГГГГ)!"
- **В день рождения**: "🎉 СЕГОДНЯ день рождения у [ФИО] (ДД.ММ.ГГГГ)! Не забудь поздравить! 🎂🎁"  
  В этот день под сообщением появляются кнопки **«Сгенерировать поздравление»** (через OpenAI) и **«Свой промпт»** (подсказка для команды `/prompt`).

💡 **Совет**: Используйте `/check` чтобы сразу проверить уведомления не дожидаясь 09:00!

### Дни напоминаний (настройка по событию)

Для каждого события можно указать, **за сколько дней** до даты напоминать:
- **0** — в сам день события (всегда по умолчанию).
- Можно указать несколько значений через запятую, например **0,1,3,7** — напоминания в день события, за 1 день, за 3 и за 7 дней.

При **добавлении** дня рождения бот спросит «За сколько дней до события напоминать?» — введите числа через запятую или `/skip` для значения по умолчанию (0,1,3,7).  
При **редактировании** (/edit) после смены даты можно изменить и дни напоминаний. Для праздников и других событий по умолчанию используется 0,1,3,7; изменить можно через редактирование записи.

### Генерация поздравлений (OpenAI)

Если задан **OPENAI_API_KEY** (в файле `openai.env` или в переменных окружения):
- **OPENAI_MODEL** (опционально) — модель для генерации. По умолчанию `gpt-4o-mini`. В `openai.env` можно указать, например: `OPENAI_MODEL=gpt-4o` или `OPENAI_MODEL=gpt-4o-mini`.
- В уведомлении «сегодня день рождения» отображаются кнопки:
  - **Сгенерировать поздравление** — бот пришлёт короткое поздравление, сгенерированное по имени именинника.
  - **Свой промпт** — бот подскажет, как отправить команду `/prompt <id> ваш промпт` для генерации поздравления в нужном стиле (например: «короткое и с юмором», «в стиле Шекспира»).
- Команда **/prompt** \<номер_записи\> \<промпт\> — сгенерировать поздравление по своему тексту. Номер записи показывается в подсказке после нажатия «Свой промпт».

## 🐳 Деплой через Caprover

### Подготовка:

1. Убедитесь, что у вас установлен Caprover CLI:
```bash
npm install -g caprover
```

2. Войдите в ваш Caprover сервер:
```bash
caprover login
```

3. Создайте приложение в Caprover dashboard (если ещё не создано)

### Настройка переменных окружения:

1. Откройте Caprover Dashboard
2. Перейдите в `Apps` → ваше приложение
3. Вкладка `App Configs`
4. Раздел `Environmental Variables`
5. Добавьте переменные:
   - **Key**: `BOT_TOKEN`, **Value**: ваш токен от BotFather
   - **Key**: `DB_DIR`, **Value**: `/app/data`
   - **Key**: `OPENAI_API_KEY`, **Value**: ваш ключ OpenAI (опционально, для генерации поздравлений)
6. Нажмите `Add` и `Save & Update`

### ⚠️ ВАЖНО: Настройка Persistent Storage для сохранения базы данных

**Без этой настройки все данные будут теряться при каждом деплое!**

1. В Caprover Dashboard → ваше приложение
2. Вкладка `App Configs`
3. Раздел `Persistent Directories`
4. Добавьте:
   - **Path in App**: `/app/data`
   - **Label** (опционально): `database`
5. Нажмите `Add Persistent Directory`
6. Нажмите `Save & Update`

Теперь все данные в папке `/app/data` (включая `birthdays.db`) будут сохраняться между деплоями!

### Деплой:

```bash
# Закоммитьте изменения
git add -A
git commit -m "Update bot"

# Деплойте
caprover deploy
# Выберите: ваш сервер → ваше приложение → main
```

### ⚠️ Важно для Caprover:

База данных `birthdays.db` создается внутри контейнера. При перезапуске контейнера данные **будут сохранены**, но при полном пересоздании контейнера данные могут быть потеряны. Для production-использования рекомендуется:
- Использовать Caprover Persistent Directories
- Или использовать внешнюю базу данных (PostgreSQL)

## 🗄️ База данных

Бот использует SQLite базу данных (`birthdays.db`) со следующей структурой:

**Таблица `birthdays`:**
- `id` - уникальный идентификатор
- `user_id` - Telegram ID пользователя
- `full_name` - ФИО человека
- `birth_date` - дата рождения (формат YYYY-MM-DD)
- `created_at` - дата создания записи

## 🔧 Технические детали

- **Библиотека**: python-telegram-bot 13.15
- **База данных**: SQLite 3
- **Планировщик**: APScheduler
- **Часовой пояс**: Europe/Moscow (MSK)
- **Время уведомлений**: 09:00
- **Формат ввода даты**: ДД.ММ.ГГГГ (15.03.1990)
- **Формат хранения**: YYYY-MM-DD (стандарт SQL)

## 🐛 Известные проблемы

- В Docker контейнере база данных не персистентна между полными пересозданиями. Используйте Persistent Directories в Caprover для production.
